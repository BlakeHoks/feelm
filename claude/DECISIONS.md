# Architecture Decision Records — Feelm

## ADR-001: Tailwind CSS вместо CSS Modules

**Контекст:** Предыдущие проекты (Watched, OfferPass) используют CSS Modules. Для Feelm решили попробовать Tailwind.

**Решение:** Tailwind CSS как единственный подход к стилизации.

**Причины:**
- Портфольный проект — хороший контекст для освоения нового инструмента
- UI-heavy проект: карточки, гриды, адаптив, тёмная тема — Tailwind ускоряет эту работу
- Расширяет резюме: CSS Modules + Tailwind показывает гибкость
- `dark:` и responsive модификаторы из коробки

**Компромисс:** Теряем привычный FSD-паттерн "стили рядом с компонентом в отдельном файле", стили уходят в JSX. Для портфолио допустимо.

## ADR-002: shadcn/ui — ограниченное использование

**Контекст:** shadcn даёт качественные accessible компоненты, но имеет узнаваемый дефолтный вид.

**Решение:** Берём ТОЛЬКО утилитарные компоненты (Slider, Dialog, Select). Всё визуально-центральное пишем сами. Тему кастомизируем до неузнаваемости.

**Причины:**
- Экономия времени на accessibility для сложных компонентов (Slider, Dialog)
- Портфолио не должно выглядеть как "ещё один shadcn-стартер"
- Карточки фильмов, MoodSelector — это витрина проекта, они должны быть авторскими

## ADR-003: Vercel Serverless Proxy вместо бэкенда

**Контекст:** Нужно скрыть TMDB API-ключ от клиента. Полноценный бэкенд — overkill для этого проекта.

**Решение:** Один файл `api/tmdb.ts` как Vercel Serverless Function.

**Причины:**
- Бесплатно (Vercel Hobby: 100k invocations/month)
- Один файл, нулевой оверхед на поддержку
- Ключ в Vercel env vars, не попадает в браузер
- Проект остаётся "фронтенд" по структуре — нет отдельного бэкенд-деплоя

**Альтернатива:** Ключ в `.env` на клиенте — для портфолио допустимо, но serverless proxy показывает зрелость подхода.

## ADR-004: Phosphor Icons вместо Lucide

**Контекст:** Lucide — стандартный выбор для shadcn, но создаёт "AI slop" эффект — все проекты выглядят одинаково.

**Решение:** Phosphor Icons.

**Причины:**
- Визуально отличается от типичных shadcn/Lucide проектов
- 6 стилей (thin, light, regular, bold, fill, duotone) — больше гибкости
- Хорошо поддерживается, tree-shakeable

## ADR-005: TanStack Router (file-based)

**Контекст:** Выбор между React Router и TanStack Router.

**Решение:** TanStack Router с file-based routing.

**Причины:**
- Type-safe routing из коробки
- Интеграция с TanStack Query (loader + prefetch)
- File-based routing убирает boilerplate
- Показывает в портфолио владение TanStack экосистемой (Router + Query)

## ADR-006: Watchlist в localStorage (Zustand persist)

**Контекст:** Нет бэкенда с авторизацией — негде хранить пользовательские данные серверно.

**Решение:** Zustand store с `persist` middleware → localStorage.

**Причины:**
- Не требует авторизации
- Мгновенный UX — нет сетевых запросов для watchlist
- Достаточно для MVP портфольного проекта

**Ограничение:** Данные привязаны к браузеру. Если проект вырастет — можно добавить Supabase Auth + DB.
